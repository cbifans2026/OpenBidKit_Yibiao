# Analytics 逻辑梳理

## 核心原则

| 项 | 核心逻辑 |
| --- | --- |
| AE | 保持现有埋点采集字段不变，保存实时/短期明细 |
| D1 | `ANALYTICS_DB` 只保存新版 `stats_*` 长期统计表，旧 `analytics_*` 表全部废弃 |
| Queue | 不使用 Queue，不保留 Queue 代码 |
| `/track` | 写 Analytics Engine；仅对 `client_created_at` 最近 3 天的新客户端尝试实时写入 D1 |
| Cron | 北京时间每天 02:00 汇总前一天 AE 数据到 D1 |
| 历史范围 | 直接读 D1，忽略 AE |
| 今天/7天/30天 | 直接读 AE，忽略 D1 |
| 客户端 ID | D1 允许保存明文 `client_id` |
| 活跃客户端 | 任意允许事件去重客户端，不限定 `app_open` |
| 公告/资源管理 | 不属于埋点统计改造，功能保持不变 |

## 存储与绑定

| 绑定 | 类型 | 用途 | 本次处理 |
| --- | --- | --- | --- |
| `ANALYTICS` | Analytics Engine dataset `agnet_analytics` | 详细埋点事件、实时查询、Cron 汇总来源 | 保持不变 |
| `ANALYTICS_DB` | D1 `openbidkit-analytics` | 长期埋点统计新版 `stats_*` 表 | 可删库重建，只保留新版表 |
| `RESOURCE_DB` | D1 `openbidkit-resources` | 资源管理元数据 | 不动 |
| `RESOURCE_BUCKET` | R2 `openbidkit` | 资源图片 | 不动 |
| `NOTICE_STORE` | KV | 公告、GitHub stats 缓存 | 不动 |

## 事件类型

| event | 统计用途 |
| --- | --- |
| `app_open` | 打开次数、留存 |
| `page_view` | 页面访问量、页面排行 |
| `config_usage` | 配置使用分布 |
| `ai_request` | 模型使用、累计 AI 请求、Token |
| `resource_click` | 资源点击统计 |

## `ANALYTICS_DB` 新版表

| 表 | 主键 | 关键字段 | 写入来源 | 查询用途 |
| --- | --- | --- | --- | --- |
| `stats_totals` | `project_name` | `total_clients`, `total_open`, `total_page_views`, `total_events`, `total_ai_requests`, `last_rollup_date` | `/track`, Cron | 概览 5 个累计总数 |
| `stats_clients` | `project_name + client_id` | `first_seen_at`, `first_seen_date`, `active_days`, `last_active_date`, `last_active_version` | `/track`, Cron | 今日/7日新增、客户端统计列表 |
| `stats_daily` | `project_name + activity_date` | `active_clients`, `app_open_count`, `page_view_count`, `event_count`, `ai_request_count` | Cron | 概览每日统计历史部分 |
| `stats_pages` | `project_name + page` | `view_count`, `client_count` | Cron | 访问分析历史页面排行 |
| `stats_versions` | `project_name + version` | 各事件计数、`client_count` | Cron | 访问分析历史版本分布 |
| `stats_configs` | `project_name + field_key + value` | `report_count`, `client_count` | Cron | 配置使用历史分布 |
| `stats_models` | `project_name + request_type + provider + endpoint_host + model` | `request_count`, token 汇总, `client_count` | Cron | 模型使用历史分布和筛选 |
| `stats_resource_clicks` | `project_name + resource_key` | `click_count`, `client_count` | Cron | 资源管理历史点击量 |
| `stats_dimension_clients` | `project_name + dimension_type + dimension_key + client_id` | `first_seen_date`, `last_seen_date` | Cron | 页面/版本/配置/模型/资源维度的长期唯一客户端去重 |
| `stats_rollup_runs` | `project_name + activity_date` | `status`, `started_at`, `completed_at`, `error` | Cron | 防止同一天重复汇总 |

## Cron 写入逻辑

| 步骤 | 查询/操作 | 写入表 |
| --- | --- | --- |
| 业务日期 | `activity_date = 昨日 Asia/Shanghai`，Cron 为 `0 18 * * *` | 无 |
| 项目发现 | AE 查询昨日 `blob1` 分组 | 无 |
| 重复保护 | `stats_rollup_runs` 中 `status='success'` 则跳过 | `stats_rollup_runs` |
| 总量 | AE 统计昨日 `app_open/page_view/ai_request/allowed events` | `stats_totals`, `stats_daily` |
| 活跃客户端 | AE 昨日任意允许事件 `COUNT(DISTINCT blob7)` | `stats_daily.active_clients` |
| 客户端生命周期 | AE 昨日按 `blob7` 分组，访问天数加 1，更新最近活跃日期和版本 | `stats_clients` |
| 页面 | AE `event='page_view'` 按 `blob3` 分组累加访问量；按 `page + client_id` 写去重关系后重算客户端数 | `stats_pages`, `stats_dimension_clients` |
| 版本 | AE 按空版本归 `未知版本` 分组累加事件量；按 `version + client_id` 写去重关系后重算客户端数 | `stats_versions`, `stats_dimension_clients` |
| 配置 | AE `event='config_usage'` 按配置字段分组累加次数；按 `field_key + value + client_id` 写去重关系后重算客户端数 | `stats_configs`, `stats_dimension_clients` |
| 模型 | AE `event='ai_request'` 按 `blob12/blob9/blob10/blob11` 分组累加请求和 token；按模型维度 + `client_id` 写去重关系后重算客户端数 | `stats_models`, `stats_dimension_clients` |
| 资源 | AE `event='resource_click'` 按 `blob9` 分组累加点击量；按 `resource_key + client_id` 写去重关系后重算客户端数 | `stats_resource_clicks`, `stats_dimension_clients` |

## `/track` 实时写入逻辑

| 步骤 | 操作 |
| --- | --- |
| 事件写入 | `env.ANALYTICS.writeDataPoint()` 写 AE |
| 客户端写入 | 仅当 `client_id` 存在且 `client_created_at` 在最近 3 天内时尝试写 `stats_clients`；同一 Worker 实例内同一客户端只尝试一次 |
| 总客户端数 | 真实插入新客户端后 `stats_totals.total_clients + 1`；老客户端不触发 D1 查询 |
| 活跃字段 | 不在 `/track` 更新 `active_days/last_active_date/last_active_version`，由 Cron 按天更新 |
| D1 失败 | 只记录 warning，`/track` 仍返回成功，避免 D1 短暂异常影响埋点入口 |

## 概览查询逻辑

| 指标 | 数据源 | 查询逻辑 |
| --- | --- | --- |
| 总客户端数 | D1 | `stats_totals.total_clients` |
| 总打开次数 | D1 | `stats_totals.total_open` |
| 总页面访问量 | D1 | `stats_totals.total_page_views` |
| 总事件数 | D1 | `stats_totals.total_events` |
| 累计 AI 请求数 | D1 | `stats_totals.total_ai_requests` |
| 今日新增客户端 | D1 | `stats_clients.first_seen_date = 今日 Asia/Shanghai` |
| 7日新增客户端 | D1 | `stats_clients.first_seen_date >= 今日往前6天` |
| 今日活跃客户端 | AE | 今日任意允许事件 `COUNT(DISTINCT blob7)` |
| 每日统计 | D1 + AE | 今日从 AE 查；前 9 天从 `stats_daily` 查 |
| 留存概览 | AE | 保持现有 `/api/retention` 逻辑，不持久化 |
| GitHub stats | GitHub + KV | 保持现有缓存逻辑 |

## 客户端统计查询逻辑

| 视图 | 数据源 | 查询逻辑 |
| --- | --- | --- |
| 客户端列表 | D1 | `stats_clients` 全量，按最近活跃日期和首次访问倒排 |
| 客户端详情 | AE | 按 `projectName + client_id` 查询 7天/30天/全部，按日期和事件分组 |

## 访问分析查询逻辑

| 范围 | 数据源 | 查询逻辑 |
| --- | --- | --- |
| 历史 | D1 | 页面读 `stats_pages`；版本读 `stats_versions` |
| 今天 | AE | 今日 `Asia/Shanghai` 过滤，页面和版本实时聚合 |
| 7天/30天 | AE | `timestamp >= NOW() - INTERVAL N DAY` 实时聚合 |
| 版本排序 | Worker JS | 语义版本倒排，`2.10.1 > 2.9.1`，`未知版本` 最后 |

## 配置使用查询逻辑

| 范围 | 数据源 | 查询逻辑 |
| --- | --- | --- |
| 历史 | D1 | `stats_configs` 按配置项和值读取累计次数和客户端计数 |
| 今天/7天/30天 | AE | `event='config_usage'` 按配置字段实时分组 |

## 模型使用查询逻辑

| 范围 | 数据源 | 查询逻辑 |
| --- | --- | --- |
| 历史 | D1 | `stats_models` 按请求类型、服务商、域名、模型读取累计请求和 Token |
| 今天/7天/30天 | AE | `event='ai_request'` 按 `blob12/blob9/blob10/blob11` 实时分组 |
| 筛选 | D1 或 AE | 服务商、域名、模型为精确筛选条件 |

## 最近事件查询逻辑

| 指标 | 数据源 | 查询逻辑 |
| --- | --- | --- |
| 总数 | AE | `COUNT()`，可按事件过滤 |
| 列表 | AE | `ORDER BY timestamp DESC, blob7 DESC, blob2 DESC, blob3 DESC LIMIT 10 OFFSET pageOffset` |
| 事件筛选 | AE | `blob2 = event`，只允许白名单事件 |

## 项目列表查询逻辑

| 优先级 | 数据源 | 查询逻辑 |
| --- | --- | --- |
| 1 | D1 | `stats_totals UNION stats_clients` |
| 2 | AE | D1 为空或失败时查最近 90 天 AE，按 `blob1` 分组 |

## 资源管理保留逻辑

| 功能 | 数据源 | 查询逻辑 |
| --- | --- | --- |
| 资源公开列表 | `RESOURCE_DB` | `resources.enabled=1`，搜索标题/标签/介绍/弹窗内容 |
| 资源后台列表 | `RESOURCE_DB` | `resources` 全量，按排序和更新时间 |
| 资源图片 | `RESOURCE_BUCKET` | `/resource-image` 读取 R2 |
| 资源点击历史 | `ANALYTICS_DB` | `stats_resource_clicks` |
| 资源点击近期 | AE | `event='resource_click'` 按 `blob9` 分组 |

## 公告与 GitHub 统计

| 功能 | 数据源 | 逻辑 |
| --- | --- | --- |
| 公告 | `NOTICE_STORE` | `project_notice:<projectName>` 读写，不属于埋点统计 |
| GitHub stats | GitHub API/HTML + KV | 30 分钟新鲜缓存，7 天 stale 缓存 |

## 接口与数据源

| 接口 | 数据源 | 核心逻辑 |
| --- | --- | --- |
| `POST /track` | AE + D1 | 写 AE，实时 upsert `stats_clients` |
| `GET /api/overview` | D1 + AE + KV | 概览总数、新增、今日活跃、每日统计 |
| `GET /api/clients` | D1 | 客户端列表 |
| `GET /api/client-detail` | AE | 单客户端日期和事件明细 |
| `GET /api/traffic` | D1 或 AE | 历史读 D1；今天/7天/30天读 AE |
| `GET /api/config-usage` | D1 或 AE | 历史读 D1；今天/7天/30天读 AE |
| `GET /api/model-usage` | D1 或 AE | 同上，支持服务商/域名/模型筛选 |
| `GET /api/latest` | AE | 最近事件分页和事件筛选 |
| `GET /api/retention` | AE | 留存概览，不持久化 |
| `GET /api/projects` | D1 优先，AE 兜底 | 项目名列表 |
| `GET/POST/DELETE /api/notice` | KV | 公告后台管理 |
| `GET/POST/DELETE /api/resources` | `RESOURCE_DB` + R2 + AE/D1 | 资源管理，保持原功能 |
| `GET /api/github-repo-stats` | GitHub + KV | 仓库统计和缓存 |
